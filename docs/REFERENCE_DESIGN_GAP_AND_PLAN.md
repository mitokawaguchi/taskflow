## デザインを寄せる際のルール（色・テイストは変えない）

- **維持する**: 現在のダークテーマ、`--accent`（#3a8f52 系）、`--critical`/`--high` 等の優先度色、フォント（Sora, Source Sans 3）、角丸・影のトーン。
- **参照に寄せる**: レイアウト（サイドバー240px・上部バー・カード余白）、カンバン列・ガント・ダッシュボードの**構成**、カード内の情報の**並び・優先度**、タブ・バッジ・「＋タスクを追加」の**配置とスタイル**（テキストリンク化など）。ラベル・アイコン・進捗バー等の**要素の有無と役割**は参照に合わせる。

## 実装計画（順次実装する場合の案）

### Phase 0：データモデル拡張（他フェーズの前提）

1. **タスクに「状態」を追加**  
   - 例: `status: 'todo' | 'in_progress' | 'review' | 'done'`  
   - 既存の `done: true` は `status === 'done'` にマッピング。既存データは `todo` または `done` に移行。
2. **タスクに「進捗率」を追加（任意）**  
   - 例: `progress: 0..100`。未指定時は状態から算出（todo=0, in_progress=50, review=80, done=100）でも可。
3. **カテゴリ（タグ）の追加（任意）**  
   - 例: タスクに `categoryId` または `tags: string[]`。マスタは「デザイン」「開発」「バグ修正」等。  
   - まずは固定ラベルでスタイルだけ寄せることも可。
4. **担当者・チーム（将来用）**  
   - 参照の「チームメンバー・アバター・メンバーフィルター」までやる場合は、`users`/`assigneeId` 等のスキーマ追加が必要。Phase 2 以降で検討。

### Phase 1：カンバンボード

1. 新ビュー「カンバン」を追加（サイドバー＋必要なら上部タブ）。
2. 4列レイアウト: 未着手 / 進行中 / レビュー中 / 完了。列ヘッダーにタスク数バッジ。
3. タスクカードをドラッグで列間移動。`status` 更新で永続化。
4. カード内: タイトル・期限・優先度（左バーは現状維持）・状態ラベル。カテゴリはデータがあれば色付きタグ表示。
5. 進捗バー（%）：データモデルに `progress` があれば表示、なければ状態から算出。
6. 各列下部に「＋ タスクを追加」（テキストリンク風スタイル、現行の色・フォントで統一）。
7. 担当者・アバターは Phase 0 でモデルを入れない場合は表示しない。

### Phase 2：ダッシュボード

1. 新ビュー「ダッシュボード」を追加。
2. 上部にサマリーカード4つ（全タスク数・進行中・完了数・期限超過数）。既存の集計ロジックを流用、カードの見た目だけ参照のレイアウトに寄せる（色は現テーマのまま）。
3. プロジェクト別進捗を一覧で表示: 既存のプロジェクト進捗率を円グラフ/ドーナツ or 横棒で集約。チャートライブラリは軽量なものを選ぶ。
4. アクティビティフィード: 「タスク完了」「タスク追加」等の簡易ログをタイムライン表示。必要なら `activity` テーブル or 既存タスクの `created`/`updated` で代用。
5. チーム負荷グラフは、担当者データを入れていない間は「プロジェクト別タスク数」などで代替するか、Phase 0 で担当者を入れてから実装。

### Phase 3：ガントチャート

1. 新ビュー「ガント」を追加。
2. 左: タスクリスト（名前・担当者（いれば）・進捗率）。右: 日付軸のタイムライン。
3. タスクの `due`（と必要なら `start`）でバーを描画。担当者別色分けはデータがあれば対応。
4. 今日の日付に縦の赤点線（`--critical` 使用）。
5. バーのホバーでツールチップ（タイトル・期限・状態等）。
6. 表示単位: 月/週/日の切替。

### Phase 4：共通UIの整理（デザイン寄せ）

1. **上部バー（参照ZIPどおり）**:  
   - 左: **現在のプロジェクト名**（または「すべてのタスク」）＋**ビュー切替タブ**（カンバン / タイムライン(ガント) / インサイト(ダッシュボード)）をピル型で表示。  
   - 右: **Filter** ボタン（既存フィルターを開く・バッジで件数）、**Search** 入力、**通知**ベル、**プロフィール**（アバター or アイコン。クリックで設定/ログインへ）。
2. フィルター: トップバーの Filter クリックで既存フィルターUIを開く。メンバーフィルターは担当者実装後にアバター横並びで追加。
3. **サイドバー**: 幅240pxは維持。**下部にチームメンバー**アバター（担当者実装前はプレースホルダー or 非表示）、**Settings**、**Logout** を追加。
4. **設定画面**: Settings クリックで設定画面（またはモーダル）を表示。テーマ切替・通知設定等のプレースホルダー。
5. **ログイン**: プロフィール or Logout からログイン/ログアウトのプレースホルダー（未実装なら「準備中」表示でも可）。
6. カードの角丸・影: 参照（8px/控えめな影）に近いが、現行の色・コントラストは維持。

### Phase 5：ラベル・期限表示・検索

1. **カテゴリ/ラベル**: タスク（および任意でPJ）にカテゴリ（デザイン/開発/バグ修正等）を追加。DB・API・フォーム・カード表示。
2. **期限表示**: タスクカード内で**フッター左**にカレンダーアイコン＋日付、超過は赤（参照TaskCard.tsx）。
3. **検索**: トップバーの Search でタスク名・メモ等を検索し一覧/カンバン/ダッシュボードを絞り込み。

## 実装順序の提案

| 順序 | 内容 | 理由 |
|------|------|------|
| 1 | Phase 0 の「状態」追加 | カンバン・ダッシュボードの前提 |
| 2 | Phase 1 カンバンボード | 参照の中心である「4列カンバン」を先に揃える |
| 3 | Phase 2 ダッシュボード | サマリー・進捗は既存集計で実装しやすい |
| 4 | Phase 0 の進捗率・カテゴリ（必要に応じて） | カードの見た目を参照に寄せるため |
| 5 | **Phase 4 構成**（トップバー・タブ・Filter・Search・通知・プロフィール・サイドバー Team・Settings・Logout・設定画面・ログインプレースホルダー） | 参照ZIPの**サイト構成・ボタン配置**を揃える |
| 6 | Phase 3 ガントチャート | 実装コストが大きいため後回し |
| 7 | **Phase 5** ラベル・期限表示位置・検索 | カテゴリ・期限UI・検索機能 |
| 8 | 担当者・チーム・メンバーフィルター | スキーマ変更が大きいため最後 |

## 補足

- **色・テイストは変えない**: 背景・文字色・アクセントは現行の CSS 変数（`--bg`, `--surface`, `--accent`, `--critical` 等）をそのまま使い、参照のライトテーマ（#F7F8FA 等）にはしない。
- **既存機能の維持**: プロジェクト一覧・「今日」「期限超過」・覚えておくこと（クライアント）・朝の確認・通知等は残す。
- 実装時は、上記 Phase ごとに小さくコミットし、動作確認とデザイン確認を並行すると安全。

## タスクテーブル（SQL）の拡張

ガント・カンバン・ダッシュボード対応に合わせて、タスクの登録項目と SQL を拡張している。

- **参照**: `docs/SUPABASE_TASKS_SCHEMA.sql`
  - タスクテーブル全体の定義（参考用 CREATE TABLE）
  - 追加カラム: `status`（カンバン）, `start_date`（ガント用開始日）, `progress`（0–100、任意）
- **既存環境**: 上記 SQL の (2)(3) を実行すると `status` / `start_date` / `progress` が追加される。
- **アプリ**: `api.js` で `start_date`・`progress` を読み書き。進捗は `progress` があればそれを表示し、なければ `status` から算出。
