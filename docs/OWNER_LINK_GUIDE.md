# 既存データのアカウント紐づけ

ログイン導入後、**今まで運用していたデータ（タスク・プロジェクト等）を、ログイン中のアカウントに紐づける**手順です。

---

## 1. 考え方

- 各テーブルに **`owner_id`**（auth.users.id への参照）を追加する。
- **既存行は `owner_id = NULL`** のまま＝「まだ誰のものでもないデータ」として扱う。
- ログイン後に **「紐づけ」** を一度実行すると、`owner_id IS NULL` の行がすべて **現在のユーザー** に紐づく。
- 紐づけ後は、そのアカウントでログインしているときだけそのデータが見える（API が `owner_id` で絞り込む）。

---

## 2. やること一覧

| 順番 | 内容 |
|------|------|
| ① | Supabase で **SUPABASE_OWNER_LINK.sql** を実行し、`owner_id` を追加する |
| ② | アプリでログインする |
| ③ | 設定またはプロフィールから **「今までのデータをこのアカウントに紐づける」** を実行する |
| ④ | 画面をリロード（または一覧を再取得）すると、紐づいたデータだけが表示される |

---

## 3. ① SQL の実行

1. Supabase ダッシュボード → **SQL Editor**
2. `docs/SUPABASE_OWNER_LINK.sql` の内容を貼り付けて実行する。
3. これで次のテーブルに `owner_id` が追加される  
   `tf_projects`, `tf_tasks`, `tf_templates`, `tf_categories`, `tf_users`, `tf_clients`, `tf_remember`

既存データはすべて **`owner_id = NULL`** のままです。

---

## 4. ② ログイン

- 既に Supabase Auth でユーザーを作成している場合  
  → そのメール/パスワードでアプリからログインする。
- まだユーザーがいない場合  
  → Supabase の Authentication → Users から「Add user」で作成するか、アプリのログイン画面でサインアップできるようにする。

紐づけは **「このアカウント」** に対して行うので、必ずログインした状態で次へ進みます。

---

## 5. ③ アプリで「紐づけ」を実行

- 画面上部の **プロフィール（👤）** をクリックして **プロフィール・ログイン** モーダルを開く。
- ログイン中は **「今までのデータをこのアカウントに紐づける」** ボタンが表示される。
- そのボタンを押すと、`owner_id IS NULL` の行がすべて **現在のユーザーの ID** で更新される。
- 紐づけ後、一覧は自動で再取得され、紐づいたデータだけが表示される。
- 何度押しても、まだ NULL の行だけが更新される（既に紐づいた行は変更されない）。

---

## 6. ④ 紐づけ後の動き

- **取得**  
  - ログイン中: `owner_id = 自分のID` または `owner_id IS NULL` のデータだけ取得する。  
    （紐づけ後は NULL がなくなるので、実質「自分のデータだけ」。）
- **新規作成**  
  - ログイン中に作ったタスク・プロジェクト等は、保存時に **`owner_id = 自分のID`** が入る。
- **未ログイン**  
  - 従来どおり `owner_id` で絞り込まないため、全データが見える（従来運用を維持）。

紐づけ後は、別アカウントでログインするとそのアカウント用のデータだけが見え、今まで運用していたデータは「紐づけたアカウント」でログインしたときだけ表示されます。

---

## 7. 注意

- **紐づけは「今ある NULL を自分の ID にする」だけ**です。  
  すでに別の `owner_id` が入っている行は変更されません。
- 本番でやる前に、**バックアップやテスト環境で一度試す**ことをおすすめします。
- RLS（Row Level Security）を有効にする場合は、  
  `owner_id` に合わせたポリシー（SELECT/INSERT/UPDATE で `owner_id = auth.uid()` など）の追加が必要です。  
  RLS を使う場合は別途ポリシーを設計してください。
